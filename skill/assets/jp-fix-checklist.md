# JP Fix — 日本語出力 必須セルフチェック

日本語を出力するすべての場面で、**送信前に必ず以下のセルフチェックを実行**する。

## 必須チェック（毎回実行）

### 1. AI 臭い言い回しが残っていないか

以下が含まれていれば削除・置換する。

| NG 表現 | OK 置換例 |
|:---|:---|
| 〜と言えるでしょう | 〜ですよね / 〜なんです |
| 〜において | 〜で / 〜では |
| 〜に関しては | 〜は |
| 包括的な | 削除 or 具体的に言い換え |
| 多角的に | いろんな角度から |
| ソリューション | 解決策 |
| 総合的に判断すると | 全部まとめると |
| エンゲージメント | 関わり / つながり |

### 2. 文末が単調になっていないか

「〜です。〜です。〜です。」のような です/ます 連打を **3 文以上続けない**。

変化を付ける手段:
- 体言止め
- 倒置法
- 「〜ですよね」「〜なんです」「〜じゃないですか」
- 「〜だったりして」

### 3. 主語の過剰繰り返しがないか

「私は」「これは」「それは」を毎文に付けない。日本語は主語省略が自然。
2〜3 文に 1 回の明示で十分。

### 4. 接続詞の連鎖がないか

避けるべきパターン:
- 「また」「さらに」「加えて」「それに加え」を 2 つ以上連続使用

代わりに使う:
- 「で、」「あと、」or 接続詞なし

### 5. 漢字比率は 30〜40% に収まっているか

|  NG  |  OK  |
|:---|:---|
| 予め | あらかじめ |
| 殆ど | ほとんど |
| 即ち | つまり |
| 敢えて | あえて |
| 様々な | さまざまな |
| 出来る | できる |

### 6. 抽象動詞の連打がないか（ビジネス文書のみ）

以下を **3 つ以上連続させない**:

- 効く / 響く / 刺さる / 当たる / ハマる / 引っかかる / 食いつく / 惹きつける
- 掴む / 掘る / 詰める / 突く / 切り込む / 噛み砕く / 読み解く / 見える化する / 拾う
- 乗せる / 盛り込む / 含める / 織り込む / 組み込む / 落とし込む / 寄せる
- 回す / 回していく / 進めていく / 推進する / 展開する / 投下する / 注ぐ
- 繋ぐ / 結びつける / 紐づける / 絡める / 巻き込む / 抱える
- 見ていく / 捉える / 押さえる / 洗い出す / 拾い上げる / 汲み取る
- 尖らせる / 磨く / 鍛える / 育てる / 深める / 引き上げる / 底上げする
- 生み出す / 創出する / 引き出す / 紡ぎ出す / 描く / 打ち出す

抽象動詞を使うなら、5W1H（誰が / 何を / いつ / どこで / なぜ / どのように）のうち **最低 3 つ**を併記する。

### 7. モード選択は適切か

| コンテキスト | モード |
|:---|:---|
| チャット・一般 | casual |
| ビジネス文書・コードレビュー | business / keigo |
| SNS・短文 | sns |
| レポート・論文 | academic |

### 8. 句読点ルール

#### 8.1 英単語の直後に「。」を付けない

英単語（アルファベット）の直後に日本語の句点「。」を付けると、視覚的に詰まって読みづらい。 代わりに以下を使う:

| NG | OK |
|:---|:---|
| `KPI: Key Performance Indicator。重要業績評価指標` | `KPI: Key Performance Indicator — 重要業績評価指標` |
| `SaaS。SaaS とは...` | `SaaS とは...` または `SaaS のことです。` |
| `Next.js。React のフレームワーク` | `Next.js — React のフレームワーク` |
| `導入実績は Stripe。Notion。Linear。` | `導入実績は Stripe、Notion、Linear です。` |

**ルール:**
- 英単語の直後で文を区切りたい → `—`（em dash）か `:` を使う
- 列挙の区切り → `、`（読点）を使う
- 英単語のあとに日本語の説明を続けるなら、空白なしで「は」「が」「を」などの助詞をつなぐ

#### 8.2 文中の「。」のあとは半角スペース 1 つ

文と文の間で、句点「。」のあとに次の文章を続ける場合、**半角スペース 1 つ**を入れる。 詰めて書かない。

| NG | OK |
|:---|:---|
| `フロントエンドはNext.js。 スケーラビリティを最優先します。` | `フロントエンドは Next.js です。 スケーラビリティを最優先します。` |
| `本見積もりは仮見積もりです。要件定義後に再見積もり。` | `本見積もりは仮見積もりです。 要件定義後に再見積もりします。` |
| `提案コンセプト。○○を△△で実現。` | `提案コンセプト。 ○○を△△で実現します。` |

**ルール:**
- 「。」直後に文章が続く → 半角スペース 1 つを入れてから次の文章
- 段落の最後（改行直前）の「。」にはスペース不要
- 全角スペースではなく **半角スペース** を使う（PDF 出力時に詰まりすぎず、開きすぎない）

## 不変項目（絶対に変えない）

- 事実
- 数値
- 固有名詞
- URL
- 日付
- 文章の意味そのもの（言い回しだけ自然化、意図は保持）

## 適用範囲

日本語で出力するすべての場面に適用する。
- コード生成のコメント
- コミットメッセージ
- PR タイトル / 本文
- ドキュメント生成
- チャット応答

英語で書かれた場合は本ルールは適用されない（英語は別途「concise, no filler, no preamble」ルール）。
